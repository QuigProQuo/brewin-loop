## CYCLE MODE: DEEP WORK

This is an **implementation cycle**. Your job is to write code — build features, solve
hard problems, make real progress on the meatiest task available.

Pick the highest-priority unchecked task from `.brewin/tasks.md` and go deep. If it's
complex, break it into pieces and start building. Stay in implementation flow — don't
context-switch to cleanup, reviews, or style fixes.

**If there are unchecked tasks, you MUST start working on one.** "No uncommitted work"
is not a reason to stop — it means you should begin the next task. Only the SHIP cycle
type wraps up a session. Your job here is to BUILD.

### What TO do:
  - Read relevant source files before coding — understand before you build
  - Write production code that advances the project
  - Run tests to verify your code works (quick feedback loop, not exhaustive audit)
  - Commit working increments as you go
  - If you discover architectural patterns while reading, note them in memory/architecture.md

### What NOT to do:
  - Do NOT do code reviews, audits, or quality sweeps — there are dedicated cycle types for that
  - Do NOT refactor or clean up code unrelated to your current task
  - Do NOT write new tests unless they directly verify code you just wrote
  - Do NOT get stuck planning — if you've read enough to start, start building

Maximize working code shipped. This cycle is measured by implementation
progress, not by issues found or test coverage added.
