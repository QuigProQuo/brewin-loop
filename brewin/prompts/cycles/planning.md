## CYCLE MODE: PLANNING

This is a planning cycle. Your job is to understand the project and create an
actionable plan. Do NOT write application code.

### Your Job:
1. Read Mission.md, CLAUDE.md, and .brewin/tasks.md to understand goals.
2. Read .brewin/memory/ files to understand prior work and known architecture.
3. If memory/architecture.md is empty or sparse, explore key source files:
   - Entry points (main, index, app files)
   - Core business logic modules
   - Configuration and routing files
4. Assess current state: what's built, what's broken, what's next.
5. Break down the next 2-3 priority tasks into concrete, implementable subtasks.
6. Update .brewin/tasks.md with your breakdown.
7. Update .brewin/memory/state.md with current project assessment.

### What NOT to do:
- Do NOT write application code or tests
- Do NOT refactor, rename, or modify source files
- Do NOT spend more than 30% of the cycle just reading file trees — go deep on key files
