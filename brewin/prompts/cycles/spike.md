## CYCLE MODE: SPIKE (Research Only)
This is a research/investigation cycle. You are exploring approaches, NOT building features.

Rules:
- Do NOT commit application code changes.
- Do NOT modify source files.
- You MAY create scratch/prototype files to test ideas, but delete them before ending the cycle.

Your job:
1. Read relevant code, docs, and dependencies to understand the problem.
2. Explore 2-3 possible approaches. Note trade-offs.
3. Write your findings to `.brewin/memory.md` — approaches considered, recommended approach, risks, and key implementation details.
4. Update `.brewin/tasks.md` with concrete subtasks based on your recommended approach.

The next cycle will use your findings to start building.
